# OPTIONS_COMPARISON

| Option | Result | Notes |
|--------|--------|-------|
| USB cable + ImageCaptureCore | **Primary** | Reliable enough for the current app. No iPhone setup beyond Trust This Computer. |
| iOS Shortcut + LAN HTTP | **Optional** | Useful when the user accepts manual Shortcut execution and local-network reachability. Setup is generated by PhoneSnap so the user does not type URL/method/headers/body. |
| QR setup / signed Shortcut install | **Optional** | Current QR points to a normal local HTTP setup page, which serves a signed generated `PhoneSnap.shortcut`. |
| GitHub Gist rendezvous | Removed | Solved changing IPs, but introduced a third-party dependency and still depended on the unreliable wireless flow. |
| iCloud Photos polling | Rejected | Latency is too high and sync behavior is outside app control. |
| AirDrop | Rejected | Manual and not automatable for the intended workflow. |
| iOS companion app + PhotoKit/App Intent | Research | Most plausible future wireless path. Needs real-device testing because iOS background execution may prevent truly automatic screenshot delivery. |

Current product direction: keep wired USB as the primary workflow, and support the local generated Shortcut as an optional wireless path without GitHub/Gist or manual Shortcut configuration.
