# OPTIONS_COMPARISON

| Option | Result | Notes |
|--------|--------|-------|
| USB cable + ImageCaptureCore | **Primary** | Reliable enough for the current app. No iPhone setup beyond Trust This Computer. |
| iOS Shortcut batch + LAN HTTP | **Optional** | Useful when the user accepts one manual Shortcut execution after taking screenshots. Setup is generated by PhoneSnap so the user does not type URL/method/headers/body. |
| QR setup / signed Shortcut install | **Optional** | Current QR points to a normal local HTTP setup page, which serves a signed generated `PhoneSnap.shortcut`. |
| GitHub Gist rendezvous | Removed | Solved changing IPs, but introduced a third-party dependency and still depended on the unreliable wireless flow. |
| iCloud Photos polling | Rejected | Latency is too high and sync behavior is outside app control. |
| AirDrop | Rejected | Manual and not automatable for the intended workflow. |
| iOS companion app + PhotoKit/App Intent | Rejected for now | Too much setup for the current product promise, and iOS background execution still does not provide a general automatic screenshot export path. |

Current product direction: keep wired USB as the primary workflow, and support the local generated Shortcut as an optional wireless batch fallback without GitHub/Gist, iCloud, app SDK setup, or manual Shortcut configuration.
